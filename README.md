# AI Unit Test Generator Agent with LangChain and Azure OpenAI

## Introduction

Testing is a critical part of the software development lifecycle. Unit tests help ensure that individual parts of an application behave correctly and prevent regressions as systems evolve. However, writing tests manually can be repetitive and time-consuming, especially for simple utility functions.

With the emergence of Large Language Models (LLMs), it is now possible to automate parts of the testing process using artificial intelligence. This project demonstrates how to build an AI-powered agent capable of generating Python unit tests automatically using the pytest framework.

The agent reads Python code and produces a test file containing pytest test cases that validate the behavior of the functions. This approach shows how AI can assist developers by automating repetitive tasks and improving productivity.

## Project Objective

The main objective of this project is to demonstrate how an AI agent can be used to automatically generate unit tests from Python code.

The agent receives Python functions as input and generates a corresponding test file following pytest conventions. The generated tests include validation of expected results as well as failure scenarios when applicable.

This project also demonstrates how to integrate modern AI development tools such as LangChain and Azure OpenAI into a practical software engineering workflow.

## Technologies Used

Python is used as the main programming language to implement the agent and the example functions.

LangChain is used to structure the interaction between the application and the language model, allowing prompts to be managed and responses to be processed programmatically.

Azure OpenAI provides the large language model responsible for analyzing the Python code and generating the corresponding test cases.

Pytest is used as the testing framework that executes the generated tests and validates the functions.

## Example Functions

This project includes simple mathematical functions such as addition, subtraction, and division. These functions are used as input for the AI agent.

The agent analyzes these functions and generates corresponding pytest tests that verify the expected behavior of each function.

## How the Agent Works

The agent reads the content of a Python file containing functions and sends the code to an Azure OpenAI model using LangChain.

A structured prompt instructs the language model to generate pytest tests that follow specific rules. The output generated by the model is saved into a new Python file containing unit tests.

These tests can then be executed using pytest to validate the original functions.

## Setup and Installation

Clone this repository.
Install the required dependencies.
Create a `.env` file based on `.env.example`.

Example configuration:
AZURE_OPENAI_API_KEY=your_api_key_here
AZURE_OPENAI_ENDPOINT=https://your-resource.services.ai.azure.com/
AZURE_OPENAI_DEPLOYMENT=gpt-4o

These variables allow the project to authenticate with Azure OpenAI.

## Running the Project

First, generate the tests automatically using the AI agent.
python generate_tests.py

This script reads the functions file and generates a pytest test file.
After the test file is generated, run the tests using pytest.

pytest
If everything is configured correctly, pytest will execute the generated tests and validate the behavior of the functions.
